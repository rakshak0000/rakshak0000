### 👥 Featured: HRMS User Personas & Solution Architecture

The HRMS product showcases how I approach **product discovery → architecture → delivery** end-to-end:

**Product Discovery — 6 User Personas** (1,00,000+ users):

| Persona | Scale | Pain Points Solved | Key Modules |
|:---|:---:|:---|:---|
| Employee (Self-Service) | 1,00,000+ | Manual HR dependency, no request visibility | Profile, Payslip, Exit, Documents |
| HR Administrator | 500–1,000 | Manual data entry, duplicate records | PIS, Onboarding, Promotion, Retirement |
| Hiring / Reporting Manager | 5,000–10,000 | Email-based approvals, no team dashboard | Approvals, Recruitment, Team Dashboard |
| Finance / Payroll Officer | 200–500 | Manual salary calculation, statutory burden | Payroll, Tax, Statutory, SAP GL Posting |
| IT / System Admin | 50–100 | Manual provisioning, scattered audit logs | IAM, RBAC, SSO, Audit Logs |
| Management / Leadership | 500–2,000 | No real-time dashboards, data silos | Dashboards, MIS, Attrition, Compliance |

**Program Orchestration — 7 User Journeys (UML Activity Diagrams):**
`Onboarding` → `Recruitment` → `Exit/Separation` → `Monthly Payroll` → `Promotion/Transfer` → `Self-Service Profile Update` → `Retirement`

**Solution Architecture:**
`React SPA (TypeScript)` → `Azure AD / Okta SSO` → `Kong API Gateway` → `Spring Boot 3.x + Apache Camel` → `Kafka` → `Redis` → `Elasticsearch` → `Workday HCM (SaaS)` + `SAP ERP (GL Posting)`
